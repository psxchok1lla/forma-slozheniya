сумматор

User Story

  • US-1:Как пользователь, я хочу ввести два числа и получить их сумму, чтобы быстро выполнять простые вычисления прямо на сайте.
  • US-2:Как пользователь, я хочу иметь возможность выбирать тему оформления сайта (тёмную или светлую), чтобы комфортно использовать интерфейс в разных условиях освещения.

Use Case

Use Case: Получение суммы двух чисел
Цель:
  Пользователь хочет получить результат сложения двух чисел.
Предусловия:
  Пользователь должен открыть сайт, где доступна форма ввода.
Акторы:
  Основной: Пользователь
В  торичный: Система
Основной поток событий:
  Пользователь вводит первое число в соответствующее поле.
  Пользователь вводит второе число.
  Пользователь нажимает кнопку «Посчитать».
  Система вычисляет сумму двух чисел.
  Система отображает результат на экране.
Альтернативные сценарии:
  При пустом вводе — считается как 0
Исключения:
  Если JavaScript отключён или нарушено выполнение скрипта, пользователь не получит результат.
  Если при расчётах требуется серверная часть (в будущем), и соединение будет недоступно — пользователь должен получить уведомление о сбое.
Постусловие:
  Результат вычислений успешно отображён пользователю.

Функциональные требования

  • FR-1: Система должна предоставлять пользователю два числовых поля для ввода значений.
  • FR-2: Система должна отображать кнопку «Посчитать» под полями ввода.
  • FR-3: После нажатия кнопки «Посчитать» система должна вычислять сумму введённых чисел.
  • FR-4: Система должна отображать результат вычисления на той же странице без перезагрузки.
  • FR-5: Пользователь должен иметь возможность выбрать тему оформления: тёмную, светлую или авто (по системной теме).
  • FR-6: Выбранная тема должна применяться мгновенно и визуально влиять на все элементы интерфейса.

Нефункциональные требования

  • NFR-1: Система должна отображать результат вычисления менее чем за 1 секунду.
  • NFR-2: Интерфейс должен быть адаптивным и корректно отображаться на экранах от 320px до 1440px.
  • NFR-3: NFR-3: Тема должна сохраняться между страницами (если будет расширение на мультистраничность).
  • NFR-4: Все элементы интерфейса должны быть доступны с клавиатуры (доступность/усв).
  • NFR-5: Система должна быть реализована с использованием HTML, CSS и JavaScript без серверной логики.
  • NFR-6: В случае непредвиденных ошибок (ввод некорректных данных, проблемы выполнения скрипта, недоступность JS) интерфейс должен предоставлять пользователю понятное сообщение.
